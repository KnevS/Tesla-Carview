Μια συλλογή ιδεών χαρακτηριστικών για μελλοντικές εκδόσεις. Εμπνευσμένη από το συνηθισμένο
σύνολο χαρακτηριστικών στον χώρο των Tesla data-tracker (Teslamate, TeslaFi,
TeslaLogger, TeslaMate forks, ABRP Companion, Watt for Tesla, κ.λπ.) —
αυτή η λίστα περιέχει **μόνο λειτουργικές περιγραφές**, χωρίς layouts UI, χωρίς
αντιγραμμένο κείμενο, χωρίς code snippets και χωρίς εισαγωγές μάρκας/ονόματος.

> "SAS Institute")· αυτό που *προστατεύεται* είναι η συγκεκριμένη έκφραση (κώδικας,
> κείμενο, γραφικός σχεδιασμός). Αυτό το έγγραφο περιορίζεται στο **τι** κάνουν συνήθως οι
> εφαρμογές σε αυτόν τον χώρο — το Tesla Carview υλοποιεί το **πώς** ανεξάρτητα.

## Συχνά στον χώρο — υποψήφια για Tesla Carview

### Ενέργεια & απόδοση

- **Κατανάλωση ανά διαδρομή έναντι WLTP** για το μοντέλο, ως δείκτης δέλτα.
- **Eco-score ανά διαδρομή** που προκύπτει από Wh/km έναντι του δικού του baseline του αυτοκινήτου
  (καθαρά τοπικό, χωρίς cloud model).
- **Θερμοχάρτης ενέργειας στη γραμμή GPS** της διαδρομής: κόκκινο = υψηλή κατανάλωση,
  πράσινο = ανάκτηση regen, κίτρινο = σταθερή ταχύτητα.
- **Προφίλ υψομέτρου** ανά διαδρομή (από open-elevation ή offline height
  tiles). Συσχέτιση κατανάλωσης ↔ μέτρων ανάβασης.
- **Διαχωρισμός climate-energy**: διαφορά μεταξύ drive-power και
  wheel-power όταν αναφέρονται και τα δύο.

### Μπαταρία & φόρτιση

- **Τάση χωρητικότητας** (καθαρά kWh με την πάροδο του χρόνου, παλινδρόμηση στη φθορά,
  "SoH 90 % αναμενόμενο περίπου MM/YYYY").
- **Συνιστώμενο παράθυρο φόρτισης** που συνδυάζει την υπάρχουσα καμπύλη τιμολογίου
  (aWattar / Tibber) με τον στόχο SoC και τον προγραμματισμένο χρόνο αναχώρησης.
- **Εκτίμηση απωλειών σε ταχεία φόρτιση** (kWh που πληρώθηκαν έναντι kWh που πραγματικά
  φτάνουν στην μπαταρία) — για οικιακή τιμολόγηση και ανά τύπο φορτιστή DC.
- **Σύγκριση καμπύλης ταχείας φόρτισης**: η τρέχουσα συνεδρία έναντι
  ιστορικών συνεδριών στην ίδια τοποθεσία / τύπο φορτιστή ως γραμμικό διάγραμμα.
- **Phantom-drain tracker**: δέλτα SoC ενώ είναι σταθμευμένο, αναλυμένο ανά
  τοποθεσία / εποχή (εκτίμηση κόστους Sentry-Mode).

### Διαδρομές & ημερολόγιο

- **Θερμοχάρτης τοποθεσιών** (συμπληρωματικός στον νέο θερμοχάρτη δραστηριότητας):
  "πού έχω βρεθεί συχνά" στον χάρτη, χωρίς γραμμές διαδρομής.
- **Ανίχνευση συχνής διαδρομής** — όταν η αρχή/τέλος είναι παρόμοιο με μια
  υπάρχουσα διαδρομή, πρότεινε ταξινόμηση (commute vs. ιδιωτική) και
  τον προηγουμένως χρησιμοποιημένο σκοπό.
- **Trip replay**: αναπαραγωγή της διαδρομής κατά μήκος της γραμμής χρόνου με
  συγχρονισμένες τιμές SoC, ταχύτητας και climate.
- **Geofence αυτο-ταξινόμηση**: πολύγωνα "Σπίτι", "Εργασία", διαδρομές
  μεταξύ τους αυτόματα tagged ως `commute`.
- **Πρότυπα επαγγελματικού συνεργάτη** (συχνά επισκεπτόμενοι
  συνεργάτες/πελάτες ως dropdown στο πεδίο εισόδου).

### Άνεση & έλεγχος

- **Αυτοματοποίηση preconditioning**: ενεργοποίηση κλιματισμού X λεπτά πριν το επόμενο
  γεγονός ημερολογίου όταν το αυτοκίνητο είναι εντός εμβέλειας και συνδεδεμένο.
- **Ευφυΐα Sentry-Mode**: auto-off στο σημείο σπιτιού μετά από X λεπτά,
  auto-on στο ξενοδοχείο/parking.
- **Συμπεριφορά πορτών**: σε προσέγγιση + phone-key + home-geofence,
  pocket-mode κλείδωμα.
- **Δυναμικό όριο φόρτισης**: στόχος SoC που συνάγεται από το αυριανό
  πρόγραμμα (προαιρετική ενσωμάτωση ημερολογίου).

### Αναφορές & ανάλυση

- **Πρόβλεψη συντήρησης** βασισμένη στην τάση χλμ (γραμμική παρέκταση):
  "Επιθεώρηση γύρω στις DD.MM.YYYY με την τρέχουσα χρήση". Μισο-υλοποιημένο μέσω
  διαστημάτων service.
- **Ρεαλισμός εμβέλειας ανά καιρό**: πραγματική κατανάλωση συσχετισμένη με
  εξωτερική θερμοκρασία (από `state.outside_temp`)· πρόβλεψη "στους -5 °C
  μια πλήρης μπαταρία είναι ~280 km σήμερα".
- **PDF ετήσιας αναφοράς**: μονοσέλιδο οπτικό με θερμοχάρτη, top 5 διαδρομές,
  σύνολο kWh, σύνολο κόστους, CO₂ έναντι ισοδύναμου diesel (υπολογισμένο τοπικά,
  χωρίς εκτίμηση cloud).
- **Λειτουργία αναφοράς CO₂**: ανά διαδρομή ένα εκτιμώμενο νούμερο CO₂ χρησιμοποιώντας
  το γερμανικό/ΕΕ grid mix ως default (ο operator μπορεί να αντικαταστήσει τα
  g/kWh, π.χ. για μερίδιο PV).

### Σύστημα & πολυχρηστικότητα

- **Family-sharing dashboard**: επιπλέον tab ανά όχημα που δείχνει "ποιος οδήγησε
  πότε" — διάγραμμα ανά οδηγό ανά εβδομάδα (χρησιμοποιεί το υπάρχον `driver_id`).
- **Κανόνες push-notification**: διαμορφώσιμα triggers (π.χ. "SoC < 20 %
  ΚΑΙ όχι στο σπίτι" → υπενθύμιση).
- **Outbound webhooks**: στόχος ανά tenant (Home Assistant, IFTTT, n8n)
  — τέλος διαδρομής, τέλος φόρτισης, λήξη συντήρησης δημοσιεύονται ως JSON.
- **Read-only API tokens** για αναλύσεις τρίτων, με επιλογή
  scope (μόνο διαδρομές / μόνο φόρτιση / μόνο μπαταρία).

### Ιδιωτικότητα & ασφάλεια (πέρα από όσα ισχύουν)

- **Λειτουργία GPS-fuzzing** ανά tenant: συντεταγμένες τελευταίου μιλίου στρογγυλοποιημένες σε
  ~200 m ώστε η ακριβής τοποθεσία σπιτιού να μην διατηρείται ποτέ (σχετικό για
  multi-driver tenants).
- **Job δικαιώματος-στη-λήθη**: διαδρομές παλαιότερες από N χρόνια
  αυτο-ανωνυμοποιούνται (στρογγυλοποίηση GPS σε 4 δεκαδικά, αφαίρεση διευθύνσεων), η εγγραφή audit
  διατηρείται.
- **WebAuthn step-up** για ενέργειες υψηλού αντίκτυπου (κατέβασμα backup,
  διαγραφή tenant) πάνω από το passkey σύνδεσης.

## Πράγματα σκόπιμα ΟΧΙ υιοθετημένα από τον χώρο

- **Εξωτερικός cloud συγχρονισμός** δεδομένων οχήματος σε dashboard τρίτου.
  Συγκρούεται με την υπόσχεση self-hosting.
- **Ιδιοκτησιακές μορφές εξαγωγής**. Διατηρούμε CSV / JSON ώστε τα δεδομένα να παραμένουν
  portable.
- **Συμμετοχή σε αναγνωρίσιμα auto-statistics** (αποστολή ανωνυμοποιημένων δεδομένων διαδρομής
  σε pool για υπολογισμό μέσων όρων μοντέλου). Μπορεί να έρθει ως opt-in αργότερα,
  ποτέ ως default.

## Προτεινόμενη σειρά προτεραιότητας

Κατάταξη ανά αξία προς προσπάθεια:

1. **Geofence αυτο-ταξινόμηση** (μικρό UI, καθημερινός αντίκτυπος)
2. **Κατανάλωση ανά διαδρομή έναντι WLTP** (ένα νούμερο, πολλή αξία)
3. **Θερμοχάρτης τοποθεσιών** (ίδιο render path με τον θερμοχάρτη δραστηριότητας)
4. **Outbound webhooks** (ανοίγει το οικοσύστημα)
5. **Ρεαλισμός εμβέλειας ανά καιρό** (χρειάζεται 1–2 εβδομάδες δεδομένων πρώτα)
6. **PDF ετήσιας αναφοράς** (σπουδαίο marketing artefact)
7. **Λειτουργία GPS-fuzzing** (σχετική για εταιρικούς tenants)

Όλες οι αναφερόμενες λειτουργίες είναι **προτάσεις, όχι δεσμεύσεις**. Κάθε
υλοποίηση γίνεται ανεξάρτητα, χωρίς λήψη εξωτερικού κώδικα ή
εξωτερικού κειμένου.
