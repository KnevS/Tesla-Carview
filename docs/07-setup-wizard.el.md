Το Tesla Carview παρέχει δύο διαδρομές για την **αρχική εγκατάσταση** και έναν ενσωματωμένο βοηθό για την **συνεχή διαμόρφωση**.

---

## Ενσωματωμένοι βοηθοί {#settings-wizard}

Από την v3.4.0 υπάρχουν **δύο ξεχωριστοί βοηθοί**:

### 1. Wizard προσωπικών ρυθμίσεων (`SettingsWizard.vue`)

Εμφανίζεται αυτόματα μετά την πρώτη σύνδεση και μπορεί να ανοίξει ξανά ανά πάσα στιγμή από **Ρυθμίσεις → 🧙 Έναρξη wizard**. Διαθέσιμο σε **όλους τους χρήστες**.

| # | Βήμα | Περιγραφή |
|---|------|-------------|
| 1 | **Καλωσόρισμα** | Επισκόπηση |
| 2 | **Γλώσσα** | Επιλογή γλώσσας εφαρμογής |
| 3 | **Σχεδιασμός** | Επιλογή στυλ σχεδίασης (Glass, Cyberpunk, Minimal, Sport, Nevs-Edition) |
| 4 | **Χρώμα accent** | Χρώμα accent για κουμπιά και πλοήγηση |
| 5 | **Μονάδες** | km/mi, °C/°F, kWh/100km κ.λπ. |
| 6 | **Dashboard** | Ορατότητα και σειρά καρτών |
| 7 | **Πλοήγηση** | Ταξινόμηση και απόκρυψη στοιχείων πλοήγησης |
| 8 | **Ειδοποιήσεις** | Εγγραφή στο Web Push, επιλογή τύπων συμβάντων |
| 9 | **Ολοκλήρωση** | Όλες οι ρυθμίσεις αποθηκεύονται |

### 2. Admin Setup Assistant (`AdminSetupWizard.vue`)

Προσβάσιμος από **Admin Hub → 🛠️ Setup Assistant**. **Μόνο για admins.** Καθοδηγεί σε όλη τη διαμόρφωση του συστήματος — χωρίς ανάγκη επεξεργασίας SSH ή `.env`.

| Βήμα | Περιγραφή |
|------|-------------|
| **Διαπιστευτήρια Tesla** | Ορισμός Client-ID, Client-Secret, Audience μέσω UI (αποθηκεύονται στη DB) |
| **Tesla OAuth** | Σύνδεση λογαριασμού Tesla (popup → PostMessage callback → auto-refresh) |
| **Οχήματα** | Συγχρονισμός οχημάτων από τον λογαριασμό Tesla |
| **Virtual Key** | Εμφάνιση/αντιγραφή URL καταχώρισης· έλεγχος κατάστασης |
| **Fleet Telemetry** | Διαμόρφωση ανά VIN· εμφάνιση κατάστασης |
| **Web Push (VAPID)** | Δημιουργία κλειδιών VAPID απευθείας στον browser ή χειροκίνητη εισαγωγή |
| **Telegram Bot** | Διαμόρφωση bot token (απαιτεί επανεκκίνηση διακομιστή) |
| **Τιμή ρεύματος** | Ορισμός χρέωσης οικιακής φόρτισης (€/kWh) ανά όχημα |
| **Εξωτερικά APIs** | Διαμόρφωση ABRP, Grok/xAI key |
| **Παρακολούθηση** | Self-healing + alert email |
| **Σύνοψη** | Επισκόπηση κατάστασης· ειδοποίηση επανεκκίνησης αν διαμορφώθηκε Telegram |

### Σημειώσεις

- **Λειτουργία πρόχειρου** (προσωπικός wizard): οι αλλαγές αποθηκεύονται μόνο στο τελευταίο βήμα
- **Άμεση αποθήκευση** (admin assistant): τα διαπιστευτήρια αποθηκεύονται βήμα-βήμα στο `tenant_settings` (DB)
- **Tesla OAuth**: παράθυρο popup· κλείνει αυτόματα μετά τη σύνδεση
- **Δημιουργία VAPID**: απευθείας στον browser — δεν χρειάζεται `docker exec`
- **Επιλογέας γλώσσας στο header** (κάθε wizard): κάθε wizard εμφανίζει έναν συμπαγή 🌐 επιλογέα πάνω δεξιά. Η επιλογή γλώσσας εφαρμόζεται αμέσως σε όλα τα κείμενα του wizard· για συνδεδεμένους χρήστες η επιλογή αποθηκεύεται στο προφίλ. Κατά τη σύνδεση εφαρμόζεται αυτόματα η γλώσσα που είναι αποθηκευμένη στο προφίλ ή το tenant default.
- **Αυτόματη αρχικοποίηση κατά την εκκίνηση του backend**: τα κλειδιά VAPID δημιουργούνται αυτόματα ανά tenant όποτε ούτε το `tenant_settings` ούτε το `.env` τα παρέχουν. Οι ειδοποιήσεις push επομένως λειτουργούν αμέσως μετά την πρώτη σύνδεση — το αντίστοιχο βήμα του wizard δείχνει "✓ ήδη διαμορφωμένο (Auto)".
- **Προ-συμπλήρωση wizard** (`GET /api/system/wizard-prefill`): όταν ανοίγει ο wizard ζητά από το backend defaults (Fleet API audience από Geo-IP, alert email = admin email, τιμή ρεύματος ανά χώρα) συν κατάσταση ανά βήμα. Ολοκληρωμένα βήματα σημειώνονται με πράσινο banner και μπορούν να παρακαμφθούν απευθείας· η οθόνη καλωσορίσματος δείχνει "X από Y βήματα ήδη ολοκληρωμένα".

Το Tesla Carview προσφέρει δύο διαδρομές για την αρχική διαμόρφωση.

## Web wizard (συνιστάται)

Στην πρώτη εκκίνηση η εφαρμογή ανιχνεύει αυτόματα ότι δεν υπάρχει ακόμα λογαριασμός διαχειριστή
και ανακατευθύνει τον browser στο **/setup**.

### Βήματα

1. **Καλωσόρισμα** — επισκόπηση συστήματος
2. **Δημιουργία λογαριασμού διαχειριστή** — επιλογή ονόματος χρήστη και ισχυρού κωδικού
3. **Ολοκλήρωση** — ανακατεύθυνση στη σελίδα σύνδεσης

Ο web wizard στο `/setup` είναι προσβάσιμος μόνο όσο δεν υπάρχει admin.
Στη συνέχεια η σελίδα κλειδώνεται αυτόματα.

## Terminal wizard

```bash
bash deploy/setup-wizard.sh
```

Ρωτά διαδραστικά:

| Παράμετρος | Περιγραφή | Παράδειγμα |
|---|---|---|
| Δημόσιο URL | Πλήρες URL της εφαρμογής | `https://tesla.example.com` |
| Tesla Client-ID | Από το Tesla Developer portal | `abc123...` |
| Tesla Client-Secret | Από το Tesla Developer portal | `xyz456...` |
| Διαδρομή βάσης δεδομένων | Αρχείο SQLite | `./data/tesla-carview.db` |
| E-mail | Για Let's Encrypt | `admin@example.com` |
| VAPID keys | Για Web Push (προαιρετικό) | άδειο = απενεργοποιημένο |

Το script γράφει τα πάντα στο `backend/.env` και ορίζει τα δικαιώματα αρχείου σε `600` (αναγνώσιμο μόνο από τον ιδιοκτήτη).

## Μεταγενέστερη αλλαγή της διαμόρφωσης

```bash
# εκτέλεση του terminal wizard ξανά (αντικαθιστά το .env):
bash deploy/setup-wizard.sh

# ή απευθείας επεξεργασία:
nano /opt/tesla-carview/backend/.env

# στη συνέχεια επανεκκίνηση των containers:
docker compose -f docker-compose.prod.yml up -d
```

## Όλες οι παράμετροι

Πλήρης λίστα μεταβλητών περιβάλλοντος στο `backend/.env.example`:

| Μεταβλητή | Υποχρεωτική | Περιγραφή |
|---|---|---|
| `PORT` | – | Θύρα backend (default: 3000) |
| `JWT_SECRET` | ✓ | Τυχαία συμβολοσειρά ≥ 64 χαρακτήρων |
| `FRONTEND_URL` | ✓ | Δημόσιο URL της εφαρμογής |
| `TESLA_CLIENT_ID` | ✓* | Tesla Fleet API client ID |
| `TESLA_CLIENT_SECRET` | ✓* | Tesla Fleet API client secret |
| `TESLA_REDIRECT_URI` | ✓* | OAuth callback URL |
| `TESLA_AUDIENCE` | – | Περιοχή Tesla API (default: NA) |
| `DB_PATH` | – | Διαδρομή προς το αρχείο SQLite |
| `ENABLE_POLLER` | – | Poller δεδομένων οχήματος on/off |
| `ADMIN_EMAIL` | – | E-mail για Let's Encrypt |
| `VAPID_PUBLIC_KEY` | – | Web Push public key |
| `VAPID_PRIVATE_KEY` | – | Web Push private key |

*Απαιτείται μόνο όταν πρόκειται να συνδεθεί όχημα Tesla.
