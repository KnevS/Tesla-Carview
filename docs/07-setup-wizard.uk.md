Tesla Carview надає два шляхи для **початкової інсталяції** та майстер всередині застосунку для **поточної конфігурації**.

## Майстри всередині застосунку {#settings-wizard}

Починаючи з v3.4.0 існує **два окремі майстри**:

### 1. Майстер персональних налаштувань (`SettingsWizard.vue`)

З'являється автоматично після першого входу і може бути знову відкритий у будь-який час через **Налаштування → 🧙 Запустити майстер**. Доступний **усім користувачам**.

| # | Крок | Опис |
|---|------|-------------|
| 1 | **Привітання** | Огляд |
| 2 | **Мова** | Вибір мови застосунку |
| 3 | **Дизайн** | Вибір стилю дизайну (Glass, Cyberpunk, Minimal, Sport, Nevs-Edition) |
| 4 | **Акцентний колір** | Акцентний колір для кнопок та навігації |
| 5 | **Одиниці** | km/mi, °C/°F, kWh/100km тощо |
| 6 | **Dashboard** | Видимість та порядок карток |
| 7 | **Навігація** | Сортування та приховування пунктів навігації |
| 8 | **Сповіщення** | Підписка на Web Push, вибір типів подій |
| 9 | **Готово** | Усі налаштування збережено |

### 2. Адмін-майстер налаштування (`AdminSetupWizard.vue`)

Доступний через **Admin Hub → 🛠️ Setup Assistant**. **Лише для адмінів.** Проводить через усю системну конфігурацію — без SSH чи редагування `.env`.

| Крок | Опис |
|------|-------------|
| **Облікові дані Tesla** | Встановити Client-ID, Client-Secret, Audience через UI (зберігаються в DB) |
| **Tesla OAuth** | Підключити обліковий запис Tesla (popup → PostMessage callback → авто-refresh) |
| **Авто** | Синхронізувати авто з облікового запису Tesla |
| **Virtual Key** | Показати/скопіювати URL реєстрації; перевірити стан |
| **Fleet Telemetry** | Налаштувати per VIN; індикація стану |
| **Web Push (VAPID)** | Згенерувати VAPID-ключі прямо в браузері або ввести вручну |
| **Telegram Bot** | Налаштувати токен бота (потребує перезапуску сервера) |
| **Тариф на електрику** | Встановити тариф домашньої зарядки (€/kWh) per авто |
| **Зовнішні API** | Налаштувати ABRP, ключ Grok/xAI |
| **Моніторинг** | Самовідновлення + email для сповіщень |
| **Підсумок** | Огляд стану; повідомлення про перезапуск, якщо було налаштовано Telegram |

### Примітки

- **Draft-режим** (персональний майстер): зміни зберігаються лише на останньому кроці
- **Негайне збереження** (адмін-майстер): облікові дані зберігаються покроково в `tenant_settings` (DB)
- **Tesla OAuth**: popup-вікно; автоматично закривається після входу
- **VAPID-генерація**: прямо в браузері — `docker exec` не потрібен
- **Перемикач мови в шапці** (кожен майстер): кожен майстер показує компактний 🌐-перемикач у верхньому правому куті. Вибір мови застосовується миттєво до всіх текстів майстра; для зареєстрованих користувачів вибір зберігається у профілі. При вході мова, збережена в профілі або tenant default, застосовується автоматично.
- **Авто-ініціалізація при старті backend**: VAPID-ключі генеруються автоматично per tenant, якщо ні `tenant_settings`, ні `.env` їх не надають. Push-сповіщення тому працюють одразу після першого входу — відповідний крок майстра показує "✓ уже налаштовано (Auto)".
- **Wizard prefill** (`GET /api/system/wizard-prefill`): при відкритті майстер запитує у backend defaults (Fleet API audience з Geo-IP, alert email = email адміна, тариф на електрику за країною) плюс статус per крок. Завершені кроки позначаються зеленим банером і можуть бути пропущені прямо; екран привітання показує "X з Y кроків уже виконано".

Tesla Carview пропонує два шляхи для початкового налаштування.

## Веб-майстер (рекомендовано)

При першому запуску застосунок автоматично виявляє, що ще немає облікового запису адміна,
і перенаправляє браузер на **/setup**.

### Кроки

1. **Привітання** — огляд системи
2. **Створити обліковий запис адміна** — обрати ім'я користувача та надійний пароль
3. **Готово** — перенаправлення на сторінку входу

Веб-майстер на `/setup` доступний лише доки немає адміна.
Після цього сторінка автоматично блокується.

## Термінальний майстер

```bash
bash deploy/setup-wizard.sh
```

Запитує інтерактивно:

| Параметр | Опис | Приклад |
|---|---|---|
| Публічна URL | Повний URL застосунку | `https://tesla.example.com` |
| Tesla Client-ID | З порталу Tesla Developer | `abc123...` |
| Tesla Client-Secret | З порталу Tesla Developer | `xyz456...` |
| Шлях до БД | SQLite-файл | `./data/tesla-carview.db` |
| Email | Для Let's Encrypt | `admin@example.com` |
| VAPID-ключі | Для Web Push (опціонально) | залиште порожнім = вимкнено |

Скрипт записує все в `backend/.env` і встановлює права на файл `600` (читання лише власником).

## Зміна конфігурації пізніше

```bash
# знову запустити термінальний майстер (перезаписує .env):
bash deploy/setup-wizard.sh

# або редагувати безпосередньо:
nano /opt/tesla-carview/backend/.env

# потім перезапустити контейнери:
docker compose -f docker-compose.prod.yml up -d
```

## Усі параметри

Повний перелік змінних середовища у `backend/.env.example`:

| Змінна | Обов'язкова | Опис |
|---|---|---|
| `PORT` | – | Порт backend (за замовчуванням: 3000) |
| `JWT_SECRET` | ✓ | Випадковий рядок ≥ 64 символів |
| `FRONTEND_URL` | ✓ | Публічна URL застосунку |
| `TESLA_CLIENT_ID` | ✓* | Tesla Fleet API client ID |
| `TESLA_CLIENT_SECRET` | ✓* | Tesla Fleet API client secret |
| `TESLA_REDIRECT_URI` | ✓* | URL OAuth-callback |
| `TESLA_AUDIENCE` | – | Регіон Tesla API (за замовчуванням: NA) |
| `DB_PATH` | – | Шлях до SQLite-файлу |
| `ENABLE_POLLER` | – | Poller даних авто увімк/вимк |
| `ADMIN_EMAIL` | – | Email для Let's Encrypt |
| `VAPID_PUBLIC_KEY` | – | Публічний ключ Web Push |
| `VAPID_PRIVATE_KEY` | – | Приватний ключ Web Push |

*Потрібно лише, коли підключатиметься автомобіль Tesla.
