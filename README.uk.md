## ⚠ Важливо — статус Tesla API станом на 2026 рік

Tesla закрила **неофіційний Owner API** для ендпоінтів автомобіля між травнем і червнем 2026 року. Раніше поширений обхідний шлях спільноти (вхід через refresh-token облікового запису Tesla з викликом `/api/1/vehicles/{id}/vehicle_data`) тепер повертає **HTTP 401 "invalid bearer token"** — обхід мертвий, і жоден патч його не оживить.

Для отримання живих даних автомобіля (батарея, клімат, TPMS, потік телеметрії) існує **лише один офіційний шлях**: Tesla **Fleet API** з ухваленням застосунку через [developer.tesla.com](https://developer.tesla.com/). Поточний час очікування — **від тижнів до місяців**.

**💡 Безкоштовний ліміт Tesla — типове приватне використання коштує 0 €/місяць:** Tesla надає **10 USD безкоштовного кредиту на акаунт щомісяця** — цього зазвичай достатньо, щоб покрити потік телеметрії для одного автомобіля та щоденні команди. Понад це — pay-as-you-go (150 000 stream-сигналів = 1 USD, 1 000 команд = 1 USD, 50 пробуджень = 1 USD). TeslaView повністю готовий — у момент схвалення вашого застосунку всі функції активуються миттєво. Очікування лише з боку Tesla; сам TeslaView завжди безкоштовний.

**Що TeslaView досі надає без ухвалення Fleet API:**

| З'єднання | Джерело даних | Що ви отримуєте | Налаштування |
|---|---|---|---|
| **OwnTracks** (рекомендовано, миттєво) | Смартфон водія | Поїздки, GPS-трек, відстань, швидкість | Майстер крок 5, ~5 хв |
| **Tesla Fleet OAuth** | Хмара Tesla | Батарея, клімат, TPMS, усе через опитування | Потрібне ухвалення Fleet API |
| **Tesla Fleet Telemetry** | Tesla → push WebSocket | Живий потік | Fleet API + Virtual Key + **реєстрація застосунку в 1 клік у майстрі** (v3.23.5) |
| **Tesla Owner API** | Хмара Tesla | ❌ **заблоковано у 2026** | — |
| **Інтеграція з Monta** | Хмара Monta | Витрати на домашню зарядку для розрахунків зі службовими авто | API-ключ у майстрі |

**Конкретно для нових інсталяцій без ухвалення Fleet:** увімкніть OwnTracks — отримаєте юридично сумісний GPS-дорожній журнал, теплову карту поїздок, облік відстані, автоматичне призначення водія. Відсутні значення батареї/клімату не є строго необхідними для класичного бізнес-журналу.

| **Шифрування у спокої** | AES-256-GCM для Tesla OAuth-токенів, секрету TOTP MFA, приватного ключа Virtual Key. Хеш + timing-safe порівняння для токенів скидання пароля. Автогенерований ключ у `data/.encryption-key` |
| **PWA з автооновленням** | Service worker виявляє розгортання й автоматично перезавантажується — без `Ctrl+Shift+R`, включно з iOS PWA |

#### Чому Telegram, а не WhatsApp / Signal?

Нас часто про це питають — короткий підсумок:

| Сервіс | Самостійний хостинг? | API для приватного бота? | Використовується тут |
|---|---|---|---|
| **Telegram** | Bot API повністю відкритий, BotFather безкоштовний, без ризику для акаунту | ✅ Так | ✅ **Так, основний канал** |
| **WhatsApp** | Лише через Meta Cloud API (бізнес-акаунт + верифікований бізнес-номер + ухвалення шаблонів). Приватне використання з власним номером **не передбачено**. Неофіційні бібліотеки (whatsapp-web.js, baileys) — це **порушення ToS** і ведуть до блокування акаунтів. | ❌ Не для приватних користувачів | ❌ **Ні** — свідомо не реалізовано |
| **Signal** | Немає офіційного бот-сервера, немає webhook API. Самостійні форки (signald) крихкі та регулярно блокуються Signal. | ❌ Ні | ❌ **Ні** |
| **Threema** | Офіційний REST API для бізнесу — але платний (~50 євро/рік за gateway-акаунт) | ⚠ Так, комерційний | ❌ Не реалізовано (платний) |
| **Web Push** (PWA) | Стандарт браузера, працює прямо на iPhone/Android, без акаунту, без сторонніх серверів окрім служби push браузера | ✅ Так | ✅ **Так, основний канал** |

**Висновок:** Telegram + Web Push разом охоплюють найважливіші канали — без сторонніх витрат, без порушень ToS, без трекінгу. WhatsApp технічно був би можливим, але налаштування (бізнес-конструкція з процесом ухвалення Meta) суперечить self-hosting-природі TeslaView. Якщо ви дійсно хочете WhatsApp: мостові рішення на кшталт *whatsapp-web.js* можуть бути додані досвідченими користувачами самостійно — ми цього не рекомендуємо.

## Багатоорендна архітектура (з v2.0)

З v2.0 Tesla Carview підтримує **кількох орендарів** з повною ізоляцією даних:

- Кожен орендар має власну базу даних SQLite
- Нові орендарі лише через **invite-посилання** з опціональною **приміткою** (Admin → Користувачі → Створити invite-посилання, дійсне 7 днів, одноразове); інвайти можна перевипустити, м'яко відкликати або видалити
- **Кілька авто** на орендаря: синхронізація через Налаштування → 🔄 Синхронізувати авто
- **Управління користувачами** на орендаря (ролі, призначення авто, блокування) з тонким налаштуванням дозволів: `Редагування авто`, `Додавання авто`, `Обов'язкове MFA` на користувача
- **Обов'язкове MFA для нових акаунтів** — router guard перенаправляє на налаштування TOTP, доки MFA не активне
- **Admin-картка завдань** перелічує активних користувачів без призначеного авто з діями в один клік
- **Записи журналу відстежують свого автора** і показують його поруч з кожним записом
- **Автентифікація через Passkey** (Touch ID, Face ID, Windows Hello, FIDO2)
- **Скидання пароля** через посилання, згенероване адміністратором
- **Визначення домашньої wallbox через Monta** (зіставлення charge-point-ID → 🏠 маркер у списку зарядки та обліку витрат)
- **Безкоштовні сесії зарядки**: позначаються в історії зарядки, виключаються з обліку витрат
- **Підвищення версії на юридичних сторінках** автоматично записує сьогоднішню дату в рядок "Stand:" перед інкрементуванням

### Таблиця обладнання для AI-режиму (локальна Ollama)

Якщо ви хочете використовувати локальний AI-чат із суверенітетом над даними (Ollama, увімкнено за замовчуванням):

| Обладнання | Рекомендована модель | RAM | tok/s (inference) | Підходить для |
|---|---|---|---|---|
| Pi 4 (4 ГБ) | `llama3.2:1b` | ~1,5 ГБ | 4–6 | прості Q&A, помітна затримка |
| Pi 4 (8 ГБ) | `qwen2.5:1.5b` | ~1,8 ГБ | 3–5 | краще, але все ще повільно |
| Pi 5 (8 ГБ) | `qwen2.5:3b` | ~3 ГБ | 4–6 | рекомендований варіант за замовчуванням |
| VPS (4 vCPU / 8 ГБ) | `qwen2.5:3b` | ~3 ГБ | 8–12 | комфортно |
| VPS / робоча станція (16 ГБ+) | `llama3:8b` | ~6,5 ГБ | 5–8 | дуже добре, трохи повільніше |
| GPU (8+ ГБ VRAM) | `llama3:8b` чи подібна | залежить від моделі | 30–80+ | корпоративний рівень |

**Вимкніть Ollama**, якщо ваше обладнання не може її запустити — створіть `docker-compose.override.yml` з:
```yaml
services:
  ollama:
    profiles: [disabled]
```
Потім `docker compose up -d` без Ollama. Або простіше: у майстрі встановіть `AI provider = Off`. Хмарна альтернатива: `AI provider = Grok` (потрібен API-ключ xAI, дані йдуть у хмару).

## Команди автомобіля та Virtual Key

Для команд автомобіля (клімат, двері, гудок тощо) необхідний **Virtual Key**.
Virtual Key дозволяє застосунку надсилати підписані команди безпосередньо автомобілю.

**Передумова**: запущений [`tesla-http-proxy`](https://github.com/teslamotors/vehicle-command) на сервері.

```bash
# запуск проксі (приклад — відкоригуйте шляхи):
tesla-http-proxy -port 4443 -host 0.0.0.0 \
  -tls-key /etc/tesla-proxy/server.key \
  -cert /etc/tesla-proxy/server.crt \
  -key-file /etc/tesla-proxy/tesla_priv.pem
```

Публічний ключ повинен бути доступний за адресою `/.well-known/appspecific/com.tesla.3p.public-key.pem`
на домені застосунку, щоб автомобіль міг перевірити ключ.

## Інтеграція з Monta (опціонально)

Tesla Carview підтримує опціональну синхронізацію з [Monta](https://monta.com) — сервісом керування зарядкою EV. Інтеграція доступна для **всіх автомобілів**:

- **Приватні авто**: сесії зарядки Monta відображаються в розрахунках як домашня зарядка (значок 🏠, автоматичне визначення домашньої wallbox).
- **Службові авто**: додатково повний кошторис витрат — щомісячний огляд, PDF-аркуш відшкодування з можливістю підпису, шаблон обліку для роботодавця.

Конфігурація на кожне авто в налаштуваннях (Профіль авто → Домашня зарядка):
- **Monta Client ID** + **Client Secret** (OAuth2, Partner API)
- **Charge Point ID** (фільтрує сесії за конкретною точкою зарядки)
- **Ціна електроенергії wallbox** (€/кВт·год, основа розрахунку для службових авто)

Синхронізація запускається вручну через **Облік витрат → Синхронізація Monta**.

## Безпека

- JWT (access-token 15 хв, refresh-token 7 днів як httpOnly cookie)
- **TOTP MFA** (Google Authenticator, Authy, 1Password тощо)
- **Passkeys** (WebAuthn, вхід без пароля)
- **10 резервних кодів** (хешованих bcrypt, одноразових)
- **Блокування акаунту** після 5 невдалих спроб (15 хв)
- **fail2ban** блокування IP після 3 невдалих входів (10 хв)
- **HTTPS** з TLS 1.2/1.3, HSTS, OCSP stapling
- Заголовки **CSP, X-Frame-Options, Permissions-Policy**
- **Rate limiting** на login та API-ендпоінтах
- **Журнал аудиту** всіх дій, релевантних для безпеки
- **Видалення даних** з попередженням про резервне копіювання та текстом підтвердження
